# Jornada do Usuário

Conforme definido no guia do trabalho, após a criação das personas, a próxima etapa é a elaboração da Jornada do Usuário para cada uma delas[cite: 658].

O objetivo desta técnica é mapear o passo a passo da interação do usuário com o produto[cite: 659], desde o momento da descoberta até a consolidação do uso. Esta análise nos permite identificar os sentimentos, as necessidades e, principalmente, as barreiras que cada perfil de usuário enfrenta em cada etapa[cite: 663, 664].

Para atender a este requisito, foi elaborada uma jornada individual para cada uma das quatro personas criadas.

### Conclusão da Etapa

A análise das jornadas foi crucial para o projeto. As **"Barreiras"** identificadas em cada perfil (como "Textos longos" para a Ana Luiza ou "Falta de feedback em tempo real" para o João Marcos) se transformaram em requisitos diretos para o nosso sistema.

[cite_start]Essa etapa foi a fonte principal de informações para a fase seguinte e final desta entrega: a **Priorização de Requisitos com MoSCoW**[cite: 694].
